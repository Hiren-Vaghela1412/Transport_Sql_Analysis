# 🚇 SQL Urban Transportation Analytics Project

## 📌 About the Project

This project focuses on analyzing an **urban transportation system database** using **SQL queries** to generate meaningful operational insights.

The dataset represents a simplified **public transport network including buses and metro services**. By structuring the data in relational tables, the project demonstrates how SQL can be used to explore **commuter behavior, route activity, vehicle performance, and revenue trends**.

This work is part of my **Data Analytics learning journey**, where I practice transforming structured datasets into actionable insights.

Key focus areas include:

* Database design and normalization
* SQL-based analytical querying
* Transportation system analytics
* Business insight generation

## 🗂️ Database Tables

The transportation system database is composed of several related tables that represent different operational components.

| Table Name        | Purpose                                                |
| ----------------- | ------------------------------------------------------ |
| commuters         | Stores passenger demographic details                   |
| stations          | Contains information about stations and zones          |
| vehicles          | Includes vehicle specifications and passenger capacity |
| routes            | Defines connections between stations                   |
| routes_assignment | Assigns vehicles to specific routes                    |
| trips             | Stores trip records including timing and fares         |
| maintenance       | Records vehicle service history and maintenance cost   |

Together these tables simulate **real-world public transportation operational data**.
